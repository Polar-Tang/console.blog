Que onda programación web?

En un principio las páginas web eran muchísimo más simple de lo que son hoy en día, lo cual sería un problema si tuvieras que hacer tu página web desde cero por las situaciones complejas en las que se puede encontrar de repente. El HTML es la estructura, las primeras páginas eran solo html, más tarde se le agregaría css para darle estilo para que no luzcan todas como wikipedia y php para el backend, aqui puedes ver más sobre la historia de php, poco más tarde Mocha, es decir JavaScript, el cuál se usa para la ejecución del código desde el

lado del cliente

y más tarde se podría utilizar también en el backend con EcmaScript5, haciendo que el backend no deba ser estrictamente en php. Sin embargo ahora se puede hacer el backend en casi cualquier lenguaje.

Conceptos básicos

HTML, CSS y JavaScript (me referire a esta triada como HTML5) destronó a php y es lo que necesitas entendes, por lo menos lo básico para poder personalizan tu página. HTML y CSS no son un lenguaje de programación como tal, sino más bien se le llama "lenguaje de etiquetas". Esto singifica que ne vez de logica complicadas, bucles y esas cosas raras que nos gusta a los programadores no está como tal, sino son serie de palabras reservedas con una sintaxis sencilla: una etiqueta se abre "<" y se cierra ">" envolviendo a dicha palabra reservada, y en la mayoria de etiquetas es común que dicha palabra se repita pero con una barra, tienen esta predisposición para que ambas etiquetas gemelas puedan envorler el contenido, pero además dentro de las etiquetas existen palabras reservadas, similares a variables sintáctimanete hablando, que son llamada atributos, y son justamente las que atribuyen cierta propiedad en está etiqueta. No explicaría todas las etiquetas porque soy un ser mortal, pero recomiendo ampliamente leer siempre la documentación oficial de JavaScript, Mdn Web Docs (en español), "Mdn huevos" si queres pronunciarlo más rápido. Pero las etiquetas más básicas son `<a href=""> </a>` es llamada anchor y href es la dirección http, en el caso nuestro contiene rutas absulutas y relativas sin el "https://" porque está dentro de la página, <link href=""></link> comunica la página con elementos externos y es usado para el css </img> con src que viene de source, es lo mismo que href, alt se utiliza para dar una descripcion, script es para introducir código de JavaScript y se usa el atributo src="" para cargar un archivo separado

Pero un momento, cargar archivos externos? no puedo meter todo adentro de la etiqueta y a seguir adelante? Pues si, la etiqueta script puede envolver código JavaScript y el estilo puede ser incluido dentro de la etiqueto con el atributo style

Pero antes que hacer esto es recomendable crear un archivo css aparte, comunicarlo con una etiqueta link y darles atributos class="" para después darles estilo y funcionalidad desde la carpeta JS. También el atributo id se puede usar con el mismo própisto, pero este no se puede repetir.
HTML5 es leído por el navegador, de arriba abajo, izquierda y derecha de forma asíncrona, es decir, cualquiera arriba de está se mostrará arriba. Esto significa que si llama un cdn con estilos y justo debajo otra etiqueta link con mis propios estilos, la segunda etiqueta sink estará sobre-escribiendo la primera


La anidación es cuando un elemento esta contenido dentro de otro, normalmente se refiere como hijo al que esta contenido dentro de una etiqueta llamada "padre". Es importante saber esto porque las etiquetas ocupan espacio, ya sea en bloque, es decir ocupando todo el ancho del padre, o en línea, ocupando el menor espacio posible, estos parrafos tiene display: inline, porque de lo contrario las etiquetas hijo ocuparían todo el ancho del padre, sería como que después de cada anchor, por ejemplo, allá un punto final, porque el anchor ocuparía todo el ancho del padre, las etiquetas p, y por lo tanto desplazaría el siguiente texto al siguiente renglón

¿Como personalizo mi blog?

Este blog utiliza Bootstrap como librería, para traer funciones de javascript predefinidas y estilos en CSS predefinidos, supuestamente para hacer el css más sencillo y menos doloroso, personalmente paso muchísimo más tiempo escribiendo el blog y haciendo el css que la parte divertida, ósea el código. Para entenderlo de una forma sencilla es como si alguien pusiera un extenso código de Js y CSS en la web y los dejara publicos, para que mediante el atributo src y href me pueda traer esas funcionalidades, ¿Pero como las funcionalidades se presentan? pues estos "archivos públicos" manipulan id's y clases, lo único que debo hacer es utilizar esas clases e ids en mis etiquetas con el link a dichos archivos presentes y gualá. Entonces la idea sería manipular todo simplemente dandole clases que son palabras reservadas.
